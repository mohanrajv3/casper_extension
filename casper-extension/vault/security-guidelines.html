CASPER Security Response Guidelines

Use this playbook when CASPER reports breach alerts or wrong-password probing activity.

Immediate Actions (First 5 Minutes)

  1. Lock the vault from CASPER UI.
  2. Open Security tab and review View Breach Details.
  3. Note service, time, source IP, and affected username.
  4. If alerts are unexpected, stop using suspicious devices/sessions.

Credential Containment (Next 15 Minutes)

  1. Change password for affected service immediately.
  2. Enable/confirm MFA on affected service account.
  3. Rotate similar/reused passwords across other services.
  4. Regenerate CASPER recovery codes and store securely offline.

CASPER-Specific Recovery

  1. In CASPER Settings, verify Honey Server URL and token are valid.
  2. Regenerate credentials if needed and confirm decoys are monitored.
  3. Run Check Breach Now to confirm no new active events.
  4. Use Show Decoys (Dev) only for controlled demos/testing.

If Wrong-Password Attempts Are Detected

  1. Treat repeated failures as possible credential probing.
  2. Check whether username is publicly exposed/reused.
  3. Increase account monitoring and rate-limiting window.
  4. Escalate if failures come from unknown repeated IP patterns.

Post-Incident Validation

  1. Ensure test email + breach email delivery is working.
  2. Confirm decoy count and monitoring status in Security Dashboard.
  3. Verify no new breach events after remediation.
